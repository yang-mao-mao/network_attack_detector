# 专题二：常见网络攻击检测系统五人分工与任务量说明

## 1. 项目基本信息

项目名称：基于流量分析与特征匹配的常见网络攻击检测系统

项目方向：专题二 - 常见网络攻击的检测系统

项目周期：2026年7月9日 - 2026年7月29日

小组成员：5人

推荐技术栈：

```text
Python + Scapy + PyQt6/Tkinter + SQLite + matplotlib/pyecharts
```

项目目标：

```text
设计并实现一个轻量级网络入侵检测系统，能够实时捕获网络流量或读取离线 pcap 文件，
解析常见 TCP/IP/HTTP 流量，根据攻击特征库和行为检测规则识别 SQL 注入、XSS、
命令执行、WebShell、暴力破解、端口扫描等常见网络攻击，并通过图形化界面进行
实时告警展示、规则管理、统计分析和报告导出。
```

## 2. 任务量评估方法

为了保证五名成员的工作量基本均衡，本项目采用三种方式综合评估任务量：

| 评估维度 | 说明 |
|---|---|
| 工作量占比 | 按项目总工作量的百分比估算，每人约 20% |
| 工作量点数 | 项目总计 100 点，每人约 20 点 |
| 交付物规模 | 按代码量、规则数量、测试样本、文档内容综合衡量 |

工作量点数说明：

```text
1 个工作量点约等于 1.5 - 2.5 小时的有效开发、测试或文档工作。
```

因此每名成员预计投入：

```text
约 30 - 50 小时有效工作量
```

考虑到成员均为大二学生，部分网络协议、抓包、规则匹配、行为检测内容需要额外学习，因此实际投入时间可能因个人基础不同略有浮动。

### 6.4 规则覆盖范围

成员 C 至少需要准备以下类型的特征规则：

| 攻击类型 | 建议规则数量 | 示例特征 |
|---|---:|---|
| SQL 注入 | 10 - 15 条 | `union select`、`or 1=1`、`sleep(`、`information_schema` |
| XSS | 8 - 12 条 | `<script`、`onerror=`、`javascript:`、`alert(` |
| 命令执行 | 8 - 10 条 | `/etc/passwd`、`;whoami`、`cmd.exe`、`powershell` |
| WebShell | 6 - 10 条 | `eval(`、`assert(`、`base64_decode`、`system(` |
| 敏感文件访问 | 5 - 8 条 | `.env`、`/proc/self/environ`、`wp-config.php` |

### 6.5 主要验收标准

```text
1. 系统可以从外部规则文件加载攻击规则。
2. 新增规则后不需要修改检测引擎代码。
3. 能够对 payload、HTTP URL、HTTP body 等字段进行匹配。
4. 命中规则后能够生成包含攻击类型、等级、证据、建议的 Alert 对象。
```

### 7.4 行为检测规则建议

| 检测类型 | 检测逻辑 | 默认阈值 |
|---|---|---|
| 端口扫描 | 同一源 IP 在时间窗口内访问同一目标多个不同端口 | 60 秒内超过 20 个端口 |
| 暴力破解 | 同一源 IP 在时间窗口内多次访问登录接口或认证服务 | 60 秒内超过 10 次 |
| 高频请求 | 同一源 IP 在短时间内发起大量 HTTP 请求 | 10 秒内超过 100 次 |
| 可疑路径探测 | 同一源 IP 请求多个敏感路径 | 60 秒内超过 10 个敏感路径 |

### 7.5 主要验收标准

```text
1. 能够检测同一源 IP 的端口扫描行为。
2. 能够检测短时间内多次访问登录接口的暴力破解行为。
3. 能够提供可重复运行的测试脚本或 pcap 样本。
4. 最终演示时能够稳定触发至少 5 类告警。
```
